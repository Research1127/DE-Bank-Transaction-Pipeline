## Using uv - Modern Python Package Manager

We'll use uv - a modern, fast Python package and project manager written in Rust. It's much faster than pip and handles virtual environments automatically.

```shell
brew install uv
```

Now initialize a Python project with uv:

```shell
uv init --python=3.13
```

This creates a `pyproject.toml` file for managing dependencies and a `.python-version` file and `uv.lock` file.

---

### File: pyproject.toml

The **`pyproject.toml`** is the central configuration file for the Python project. It is the modern standard (defined by [PEP 518](https://peps.python.org/pep-0518/) and [PEP 621](https://peps.python.org/pep-0621/)) that replaces older files like `setup.py` and `setup.cfg`.

It typically contains:

- **Project metadata** — name, version, description, authors, license, Python version requirement.
- **Dependencies** — the packages your project needs to run.
- **Dev dependencies** — tools only needed during development (e.g., linters, test runners).
- **Build system** — specifies the backend used to build/package the project (e.g., `uv`, `hatchling`, `setuptools`).
- **Tool configuration** — settings for tools like `ruff`, `mypy`, `pytest`, etc., all in one place.

```toml
[project]
name = "my-app"
version = "0.1.0"
requires-python = ">=3.11"
dependencies = ["fastapi", "httpx"]

[tool.uv]
dev-dependencies = ["pytest", "ruff"]
```

---

### File: uv.lock

The **`uv.lock`** is an auto-generated lockfile created by [**uv**](https://github.com/astral-sh/uv), a fast Python package manager. It should **never be edited manually**.

Its purpose is to:

- **Pin exact versions** of every dependency (including transitive ones) so that every team member and every deployment installs the exact same packages.
- **Ensure reproducibility** — eliminates "works on my machine" issues.
- **Speed up installs** — `uv` resolves dependencies once and stores the result here.

```toml
# This file is auto-generated by uv. Do not edit manually.
[[package]]
name = "fastapi"
version = "0.111.0"
source = { registry = "https://pypi.org/simple" }
```

> ✅ **Commit `uv.lock` to version control** so your entire team and CI/CD pipeline uses identical dependency versions.  
> ❌ **Do not edit it manually** — always let `uv` regenerate it via `uv lock` or `uv sync`.

## Install dependencies as needed:

Follow instruction in Kaggel API

```shell
uv add "kagglehub[pandas-datasets]"
```

Add pyarrow for parquet

```shell
uv add pyarrow
```

Run the ipynb script or python script, but make sure adjust the file_path following dataset name in our Kaggle Page

### File: pipeline.py

```python
# Install dependencies as needed:
# pip install kagglehub[pandas-datasets]
import kagglehub
from kagglehub import KaggleDatasetAdapter

# Set the path to the file you'd like to load
file_path = "bank_transactions_data_2.csv"

# Load the latest version
df = kagglehub.load_dataset(
  KaggleDatasetAdapter.PANDAS,
  "valakhorasani/bank-transaction-dataset-for-fraud-detection",
  file_path,
  # Provide any additional arguments like
  # sql_query or pandas_kwargs. See the
  # documenation for more information:
  # https://github.com/Kaggle/kagglehub/blob/main/README.md#kaggledatasetadapterpandas
)

df.to_csv("/output/Bank_Data.csv")
df.to_parquet("/output/bank_transactions.parquet", index=False)
```
